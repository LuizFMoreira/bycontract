## 📝 Sobre este Trabalho

Este repositório contém o resumo crítico do artigo acima, elaborado como atividade avaliativa da disciplina de **Projeto de Software**.

O resumo aborda os seguintes tópicos extraídos do artigo:

- A metáfora central: software como sucessão de contratos documentados
- Pré-condições, pós-condições e invariantes de classe
- Crítica fundamentada à programação defensiva
- Princípios para um mecanismo de exceções seguro (rescue e retry)
- Herança e dynamic binding interpretados como subcontratação
- A regra de redefinição de métodos (precursora do Princípio de Liskov)
- Defesa do uso de rotinas parciais bem documentadas
- Reflexão sobre a relevância contemporânea dos conceitos apresentados
